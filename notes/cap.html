CAP Theorem

Definitions

  • Consistency. Ensuring that all clients see the same data at the same time
  • Availability. Ability of a system to respond to requests
  • Partition Tolerance. Ability of a system to operate if there is a network partition (for example, due to network failures)

Theorem

If the system prioritises consistency, it may become unavailble until the partition is resolved. If the system prioritises availability, it may allow updates to the data resulting in data inconsistencies.

Consistency, Availbility & Partition Tolerance cannot be supported at the same time by a distributed system.

If a partition happens, the system must choose between consistency and availbility.

In real world, there is a degree of availbility and consistency which has to be kept in mind before designing a distributed system.
